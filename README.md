## 搬瓦工服务器迁移是什么，能解决什么问题

搬瓦工的机房迁移（官方叫 Migrate to another DC），本质是把你 VPS 上的所有数据原封不动搬到另一个数据中心。迁移期间服务器仍然可以操作，只在最后阶段会关机几秒到几分钟；迁完之后 IP 会换成目标机房的新 IP，你的系统、文件、配置全部完整保留。

用这个功能最常见的几种场景：

- 买的时候默认在洛杉矶 DC2 普通线路，晚高峰卡顿，想换到 CN2 或 CN2 GIA 机房
- IP 被封，需要换机房获取新 IP（注意：IP 被封的情况下无法直接换机房，需先付费更换 IP）
- 想测试不同机房的实际速度，哪个好用就留哪个
- 之前在美国机房，现在业务方向变了，想换到日本或香港

## 搬瓦工服务器迁移操作步骤

整个流程不复杂，按顺序来。

**步骤一：登录 KiwiVM 控制面板**

进搬瓦工官网，点上方 Services → My Services，会列出你所有的 VPS 实例。找到要迁移的那台，点右侧的「KiwiVM Control Panel」按钮直接进入。

**步骤二：关机**

进面板后，先在首页找到「Stop」按钮，把 VPS 关掉。迁移前必须先关机，不关机直接迁的话系统会提示你先停机。

**步骤三：进入迁移功能**

左侧菜单往下拉，找到「Migrate to another DC」，点进去。

**步骤四：选择目标机房**

页面上会显示当前机房，下方列出这个套餐支持的所有可迁移机房。从列表里选你想去的那个，点「Confirm on next step」。

**步骤五：确认并开始迁移**

下一个页面会显示确认信息，包括迁移后 IP 会变更的提示。确认没问题，点「Start Migration」，等待完成。

迁移时间取决于 VPS 上的数据量，一般 10-30 分钟以内。迁完后你注册邮箱会收到一封确认邮件，里面有新的 IP 地址。

## 迁移完成后要做的事

三件事不能忘。

1. **更新 DNS 解析**。如果你的 VPS 上跑着网站，域名解析的 IP 还是旧的，网站会无法访问。迁移完后马上去域名商那边改 A 记录，指向新 IP。

2. **检查 SSH 连接**。新 IP + 原来的端口和密码，重新连一次，确认能登进去。如果连不上，可以先在 KiwiVM 面板用 Root Shell 功能检查一下状态。

3. **IPv6 地址要重新设置**。切机房不会自动更新 IPv6，如果你之前启用了 IPv6，建议在 KiwiVM 里先删掉再重新添加，不然延迟会偏高。

顺便一说，迁移前最好先做个快照（KiwiVM 左侧菜单里的 Snapshots），万一迁移过程中出了什么问题，还有个退路。搬瓦工每台 VPS 可以永久保存 2 个快照，免费的，不用白不用。

## 各套餐支持迁移的机房范围

这是最多人搞混的部分。不同套餐可以迁移的机房范围不一样，不是买了就能随便换到任何机房。

**KVM 常规套餐**：默认在 DC2 洛杉矶，可以切换到美国洛杉矶 DC3/DC4/DC8、弗里蒙特、新泽西、纽约、荷兰阿姆斯特丹，共 9 个机房。注意 DC3 是 CN2 GT 线路，切过去流量会变成三分之一。

**CN2 套餐（洛杉矶 CN2）**：和 KVM 常规套餐大致相同，可以在几个普通机房和 CN2 机房之间来回换。

**CN2 GIA-E 套餐**：这个是搬瓦工机房迁移功能体验最好的套餐，可以切换到洛杉矶 DC6/DC9（CN2 GIA 线路）、日本大阪软银（JPOS_1）、荷兰联通 9929（EUNL_9）、加拿大温哥华 CN2 GIA（CABC_6）等，多达 12 个以上高速线路机房。

**香港 CN2 GIA 套餐**：独立套餐，不支持从其他套餐迁入。买了香港套餐，可以切换到 HKHK_1（PCCW 线路），但不能把 CN2 GIA-E 套餐迁进香港机房。

迁移方向也有规则：同级别机房之间可以互迁，高级别可以迁到低级别，但低级别不能迁往高级别机房。

## 迁移的流量消耗问题

这点常被忽略。迁移过程本身会消耗流量——搬到新机房的数据传输量会计入你当月的流量配额。如果 VPS 上装了很多东西，建议迁移前清理一下不用的文件，减少传输量。

流量快用完的时候迁移还有另一个问题：迁移本身消耗的流量可能导致超额。搬瓦工流量超出后 VPS 会被暂停，所以时机选在流量充裕的时候比较好。

## 迁移失败或迁不了，常见问题处理

**Q：找不到「Migrate to another DC」选项**

这个套餐可能本身不支持机房迁移，或者套餐下的机房是固定的（比如 MINICHICKEN 限量版绑定弗里蒙特机房，无法迁移）。进面板看看有没有这个菜单项，没有就是不支持。

**Q：提示 IP 被封，无法迁移**

IP 被封后，KiwiVM 里的迁移功能会被锁住。需要先在面板里付费更换一个新 IP，换好之后再做机房迁移。

**Q：选了目标机房，但列表里没有我想要的机房**

每个套餐的可迁移机房是固定的，不是所有机房都能选。CN2 GIA-E 套餐可以选的机房最多，如果你现在的套餐能迁到的机房有限，本质上是套餐级别的问题，不是操作问题。

**Q：迁移了半天还没完成**

数据量大的话可能需要一两个小时，属于正常情况。等就行，完成后会发邮件通知，不用一直盯着面板。

## 常见问题 FAQ

**Q：搬瓦工服务器迁移收费吗？**

不收费。机房迁移完全免费，次数也没有限制，官方没有规定迁移频率的上限。

**Q：迁移后数据会丢失吗？**

不会。迁移过程全程保留数据，系统、文件、配置都原样复制到新机房。为了稳妥，建议迁移前先做一个快照。

**Q：迁移后 IP 变了，我的域名怎么办？**

需要手动更新 DNS 记录，把 A 记录指向新 IP。IP 换了域名解析不更新的话，网站会无法访问。更新 DNS 后，解析生效通常需要几分钟到几小时，视 TTL 设置而定。

**Q：买哪个套餐机房迁移最灵活？**

CN2 GIA-E 套餐可切换的机房最多，12 个以上高速线路机房任意选，季付 $49.99 门槛也不高，是机房迁移体验最好的套餐。

**Q：迁移一次需要多长时间？**

通常 10-30 分钟，数据量大的话可能到一两小时。完成后会收到邮件通知。
